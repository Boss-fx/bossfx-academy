- **Method:** Email + password (Supabase Auth)

## How It Works

### Backend Access (lib/supabase.js)
```javascript
const { createClient } = require('@supabase/supabase-js');
const supabase = createClient(SUPABASE_URL, SUPABASE_SERVICE_ROLE_KEY);
```
The service_role key bypasses RLS, giving full CRUD access. Used by all `/api/` endpoints.

### Client Access (admin/index.html)
```javascript
const supabase = window.supabase.createClient(SUPABASE_URL, SUPABASE_ANON_KEY);
```
The anon key is subject to RLS policies. Used only by the admin dashboard for auth and reading data.

### File Upload (admin scripts)
Product files are uploaded via admin CLI scripts:
```bash
node scripts/upload-product-file.js forex-101 ./path/to/course.pdf "Forex 101 Course"
```

### Signed URL Generation (lib/files.js)
```javascript
const { data } = await supabase.storage
    .from('product-files')
    .createSignedUrl(fileKey, 300); // 300 seconds = 5 minutes
```

## Common Failures

| Failure | Cause | Fix |
|---|---|---|
| "Database unavailable" in API responses | `SUPABASE_URL` or `SUPABASE_SERVICE_ROLE_KEY` not set | Add env vars to Vercel |
| RLS policy violation | Using anon key for write operations | Backend must use service_role key |
| Storage URL expired | Signed URL older than 300 seconds | Generate new signed URL (token still valid) |
| Admin dashboard won't load data | `SUPABASE_ANON_KEY` not in HTML meta tag | Check admin/index.html meta tags |
| "duplicate key violation" on order insert | Order already exists (flw_transaction_id constraint) | Expected behavior — dedup working correctly |

## Recovery Steps

### Database Connection Issues
1. Check Supabase dashboard status (status.supabase.com)
2. Verify env vars are set: `vercel env ls`
3. Test connection via `/api/health`
4. Check Supabase project hasn't been paused (free tier pauses after inactivity)

### Missing Product Files
1. Check `product_files` table for the product_id
2. Verify file exists in `product-files` bucket (Supabase Dashboard → Storage)
3. Upload file: `node scripts/upload-product-file.js <product-id> <file-path> "<display-name>"`

### Reactivating Paused Project
Free-tier Supabase projects pause after 7 days of inactivity. To reactivate:
1. Go to Supabase Dashboard
2. Select the project
3. Click "Restore Project"
4. Wait 1-2 minutes for the database to come back online

## Key Constraints

- **Free tier:** 500MB database, 1GB storage, 2GB bandwidth, project pauses after 7 days inactivity
- **No direct SQL in dashboard:** Use the SQL editor for schema changes, but ALTER TABLE is limited
- **JSONB workaround:** Use the `meta` JSONB column on orders to extend data without schema migration
- **No automated backups:** Free tier doesn't include point-in-time recovery (planned in Phase 4)
